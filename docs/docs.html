Getting Started

Installation

Almagest requires Python 3.11+. Install it from source into a virtual environment:

shell
git clone https://github.com/jkbarrerab/almagest.git
cd almagest
python3.11 -m venv .venv && source .venv/bin/activate
pip install -e .

After installation almagest is available on your PATH inside the virtual environment. To call it from any directory, add the .venv/bin/ folder to your shell profile:

~/.zshrc or ~/.bashrc
export PATH="/path/to/almagest/.venv/bin:$PATH"

Getting Started

Quick start

Run any workflow by passing a topic or bibcode as the first argument. All outputs are written to outputs/ as Markdown files.

shell
# Literature review
almagest lit "AGN feedback quenching"

# Deep multi-agent research brief
almagest deepresearch "JWST constraints on galaxy formation at z > 10"

# All papers about a specific object
almagest source "NGC 1068" --topic "integral field spectroscopy"

# Simulated peer review
almagest review 2023ApJ...950...72C

# Monitor a topic for new papers
almagest watch add "stellar feedback dwarf galaxies"
almagest watch run --digest
Getting Started

Extra context

Every workflow accepts --context and --context-file to inject additional instructions, bibcodes, or constraints directly into the agent's prompt before it starts.

Inline text
almagest lit "AGN feedback" --context "Focus on low-mass galaxies (M* < 10^10 Msun) and observational evidence only"
From a file
almagest review --pdf paper.pdf --context-file my_notes.md

Use context to steer the research direction, cite specific papers you want considered, restrict the scope, or ask the agent to pay particular attention to a method or claim.

Workflow

deepresearch

deepresearch is the flagship Almagest workflow. It dispatches multiple researcher agents in sequence to search NASA ADS and the web from different angles, then synthesizes everything into a structured research brief with inline citations and a full BibTeX bibliography.

Usage

shell
almagest deepresearch "dark matter halo formation at high redshift"
almagest deepresearch "mechanisms of AGN feedback quenching star formation"
almagest deepresearch "JWST constraints on galaxy formation at z > 10"

# Steer the investigation with extra context
almagest deepresearch "AGN feedback quenching" \
  --context "Focus on low-mass galaxies (M* < 10^10 Msun) and observational evidence only"

How it works

  • 01
    Search

    Multiple ADS queries are issued from different angles — by keyword, author, review articles, citation-ranked, and date-ranked — to ensure broad coverage of the topic.

  • 02
    Extraction

    Abstracts, metadata, and citation counts are fetched for the most relevant papers. The agent traces key claims back to specific bibcodes.

  • 03
    Synthesis

    Findings are cross-referenced and organized into a structured research brief: background, key findings, open questions, and a BibTeX bibliography.

  • 04
    Citation

    Every factual claim is labeled with its source bibcode. Claims that cannot be confirmed are marked [UNVERIFIED] or [INFERRED].

Output

  • SummaryA concise overview of the topic and the state of the field.
  • BackgroundHistorical context, foundational papers, and the evolution of the field.
  • Key FindingsThe main results and claims from the literature, each with inline citations.
  • Open QuestionsActive debates, unresolved tensions, and gaps in the current evidence.
  • BibliographyA complete BibTeX block for all cited papers, ready to paste into your manuscript.

Saved to: outputs/<slug>-deep-research.md

Customization

You can steer the research by being specific in your prompt. Narrow topics produce focused, evidence-dense briefs. Broad topics produce survey-style overviews. Use --context to restrict scope, specify a redshift range, prioritize certain instruments, or ask the agent to include specific papers you already know about.

Workflow

lit

lit produces a systematic literature review from primary sources: foundational papers, key milestones, the current consensus, active debates, and open gaps. It is designed as background research before starting a new project or writing a paper introduction.

Usage

shell
almagest lit "stellar feedback in dwarf galaxies"
almagest lit "fast radio bursts progenitor models"
almagest lit "integral field spectroscopy IFU surveys"

# Include specific papers you want covered
almagest lit "oxygen abundance gradients" \
  --context "Include IFU surveys only. Make sure to cover:
  - 2014A&A...563A..49S (CALIFA survey)
  - 2018MNRAS.481.2299S (MaNGA gradients)"

How it works

The agent issues multiple ADS queries with different strategies — most-cited papers for seminal work, doctype:review for review articles, date-ranked results for recent developments, and abstract-keyword searches for methodological coverage. It fetches full metadata and abstracts for the top papers, then synthesizes findings into a structured review that maps the consensus and flags active debates.

Output

  • IntroductionDefinition of the topic and why it matters.
  • Foundational WorkSeminal papers that established the field, with citation counts.
  • Current ConsensusWhat is well established and broadly agreed upon.
  • Active DebatesConflicting results, competing models, and ongoing controversies.
  • Open QuestionsWhat remains unknown, poorly constrained, or underexplored.
  • BibliographyBibTeX for all cited papers.

Saved to: outputs/<slug>-lit-review.md

Customization

Restrict the review to a specific time window ("Focus on work published after 2015"), a particular instrument ("CALIFA and MaNGA only"), or a redshift range. You can also seed it with bibcodes you want included and ask the agent to explain how they relate to the overall narrative.

Workflow

source

source searches NASA ADS for all literature about a specific astronomical object or sky position, resolves the name via SIMBAD/NED, and synthesizes a complete object overview with a discovery timeline, instrument coverage, and consensus findings.

Usage

By object name
almagest source "NGC 1068"
almagest source "M87" --topic "jet kinematics"
almagest source "Crab Nebula" --topic "pulsar wind nebula"
By sky position
almagest source --ra 40.669 --dec -0.014 --radius 5arcmin
almagest source --ra 187.706 --dec 12.391 --radius 2arcmin --topic "radio jet"
Radius formats — accepted values: 5arcmin, 30arcsec, 0.5deg, or a plain number (treated as arcmin). Maximum radius is 1 degree.

How it works

The agent first runs a broad ads_search_object call to find all ADS papers about the object (resolved via SIMBAD/NED). If a science topic is specified, a second filtered call narrows the result set. For positional queries, a cone-search is performed using ADS's position resolver. The agent then fetches full abstracts for the top 8–10 papers and synthesizes a structured report.

Output

  • Object OverviewObject type, distance, redshift, and notable characteristics with the defining paper.
  • Papers FoundTotal count in ADS and count matching the topic filter.
  • Key PapersTable of the top 10–15 papers with bibcode, authors, year, topic, and citation count.
  • Discovery TimelineChronological list of key findings and observations, with instrument and bibcode.
  • Instrument CoverageWhich telescopes and surveys have observed this source.
  • Consensus & Open QuestionsWhat is well-established vs. still debated.
  • BibliographyBibTeX for all cited papers.

Saved to: outputs/<slug>-source-lit.md

Customization

Use --topic to filter results to a specific science topic (e.g. "integral field spectroscopy", "AGN", "star formation"). Use --context to restrict to a specific epoch or instrument: "Focus on MUSE and SINFONI observations published after 2015".

Workflow

review

review simulates a peer review of a paper. It accepts an ADS bibcode, a local PDF, or both, and produces a structured review with severity-graded issues and an actionable revision plan.

Usage

shell
# Review by ADS bibcode
almagest review 2023ApJ...950...72C

# Review a local PDF (preprint, paywalled paper, or your own manuscript)
almagest review --pdf ~/Downloads/my_paper.pdf

# Both: full PDF text + ADS metadata and related-work search
almagest review 2023ApJ...950...72C --pdf ~/Downloads/paper.pdf

# Include papers you want the reviewer to consider
almagest review --pdf paper.pdf \
  --context "Consider these related papers:
  - 2021ApJ...910...72C  (metallicity gradients, same method)
  - 2019MNRAS.484.5230R  (IFU methodology reference)
  Pay attention to whether systematic errors in abundance calibration are propagated."

How it works

The agent reads the full paper text (from the PDF or ADS abstract) and evaluates it against standard peer review criteria: novelty, methodology, statistical rigor, discussion of limitations, and citation completeness. It then searches ADS for the most relevant related work to assess whether the authors have missed important references. Each issue found is tagged with a severity level.

Output

  • SummaryOne-paragraph assessment of the paper's contribution and overall quality.
  • StrengthsWhat the paper does well.
  • IssuesGraded list: CRITICAL · MAJOR · MINOR · COSMETIC.
  • Missing CitationsRelevant papers found in ADS that were not cited.
  • Revision PlanNumbered action list of changes recommended before publication.
  • RecommendationAccept / Minor revision / Major revision / Reject, with rationale.

Saved to: outputs/<slug>-peer-review.md

Customization

Point the reviewer toward specific concerns with --context: ask it to focus on the statistical treatment, compare the methodology to a specific reference, or check whether a claim is supported by a particular paper. Provide a --context-file with your own notes to make the review reproducible across sessions.

Workflow

audit

audit checks whether any code repository linked to a paper (GitHub, Zenodo, ASCL) actually reproduces the methods, parameters, and results described in the text. It flags mismatches between the paper and the code.

Usage

shell
almagest audit 2022ApJ...935..167V
almagest audit 2019JOSS....4.1414F

# Focus the audit on a specific claim
almagest audit 2022ApJ...935..167V \
  --context "Check whether the stellar mass function parameters in Table 2
  match the values hardcoded in the fitting routine."

How it works

The agent fetches the paper's metadata and abstract from ADS, then searches the linked data and software identifiers for a public code repository. It clones or inspects the repository, compares key numerical values (parameters, thresholds, constants) between the paper and the code, and checks that the described analysis steps can be traced to actual code paths. Each discrepancy is reported with the relevant line numbers or file paths.

Output

  • Paper SummaryKey methods, parameters, and results as described in the text.
  • Code FoundRepository URL and description of what the code does.
  • MatchesClaims that are correctly reproduced in the code.
  • MismatchesDiscrepancies between the paper and the code, with locations.
  • MissingSteps described in the paper with no corresponding code.
  • VerdictOverall reproducibility assessment.

Saved to: outputs/<slug>-audit.md

Workflow

replicate

replicate generates a step-by-step plan to independently reproduce the key results of a paper, covering data access, software environment, expected intermediate outputs, and validation checks. Add --execute to run the first steps in a sandboxed shell.

Usage

shell
# Plan only (safe — no code runs)
almagest replicate 2020MNRAS.499..230B

# Plan + execute (clones repos and runs scripts)
almagest replicate 2020MNRAS.499..230B --execute

# Provide environment constraints
almagest replicate 2020MNRAS.499..230B \
  --context "I only have access to public SDSS data. Skip any steps requiring
  proprietary datasets. Target Python 3.11."

Output

  • Data AccessWhere to download the required datasets and any access restrictions.
  • EnvironmentRequired packages, versions, and setup commands.
  • StepsNumbered replication plan from raw data to final figures.
  • Expected OutputsWhat each step should produce, with validation criteria.
  • CaveatsPotential obstacles: missing data, stochastic components, hardware requirements.

Saved to: outputs/<slug>-replication.md

Workflow

compare

compare places competing theories, methods, or results side by side and produces an agreement and conflict matrix. Useful for understanding where the literature converges and where it disagrees.

Usage

shell
almagest compare "NFW vs Einasto dark matter density profiles"
almagest compare "ram pressure stripping vs AGN feedback as quenching mechanism"
almagest compare "photometric vs spectroscopic redshift accuracy for survey science"

# Guide the comparison dimensions
almagest compare "NFW vs Einasto dark matter profiles" \
  --context "Focus on: (1) fit quality in dwarf galaxies,
  (2) behaviour at small radii, (3) N-body simulation results post-2015."

Output

  • OverviewBrief description of each approach being compared.
  • Agreement MatrixDimensions on which the sources agree, with supporting citations.
  • Conflict MatrixDimensions on which the sources disagree, with the evidence on each side.
  • VerdictCurrent state of the debate: resolved, ongoing, or context-dependent.
  • BibliographyBibTeX for all cited papers.

Saved to: outputs/<slug>-comparison.md

Workflow

draft

draft researches a topic from NASA ADS and produces a full paper-style draft — abstract through conclusions — with inline citations and a BibTeX bibliography. It is not submission-ready, but is a solid starting point that accelerates the writing process.

Usage

shell
almagest draft "the role of environment in galaxy quenching at z ~ 0.5"
almagest draft "a review of chemical abundance gradients in nearby galaxies"

# Seed the draft with your own data and target journal
almagest draft "chemical abundance gradients in nearby galaxies" \
  --context "Assume we have MaNGA IFU data for 500 galaxies.
  Emphasise comparison with CALIFA results. Target audience: ApJ."

Output

  • Abstract200-word structured abstract.
  • IntroductionMotivation, context, and statement of the problem.
  • Background / MethodsReview of relevant prior work and proposed methodology.
  • Results & DiscussionKey findings with inline citations and interpretation.
  • ConclusionsSummary and future directions.
  • BibliographyComplete BibTeX block.

Saved to: outputs/<slug>-draft.md

Customization

Describe your data, instruments, or preliminary results in --context so the draft reflects your actual situation rather than a generic treatment. Specify a target journal to adjust the writing style and reference conventions.

Workflow

autoresearch

autoresearch starts from a seed idea and runs an autonomous loop: form a hypothesis, search ADS for evidence, evaluate support, refine the hypothesis, and repeat. It continues until the evidence converges or --max-rounds is reached.

Usage

shell
# Default 5 rounds
almagest autoresearch "galaxy quenching driven by AGN feedback"

# More thorough
almagest autoresearch "origin of the stellar mass-metallicity relation" --max-rounds 10

# Quick exploration
almagest autoresearch "tidally stripped stars as probes of dark matter" --max-rounds 3

# Constrain the hypothesis space
almagest autoresearch "galaxy quenching at cosmic noon" \
  --context "Restrict to observational evidence at 1.5 < z < 2.5.
  Prioritise JWST and ALMA results published after 2022."

How it works

Each round the agent (1) states its current best hypothesis, (2) searches ADS for papers that could confirm or refute it, (3) evaluates the evidence and updates its confidence score, and (4) refines the hypothesis for the next round. The final report records each hypothesis, the evidence considered, and the confidence trajectory across rounds.

Note: This workflow performs multiple ADS search rounds and can consume significant API quota. Use --max-rounds 3 for quick explorations.

Output

  • Hypothesis logThe hypothesis at each round, with the evidence considered.
  • Evidence summaryPapers supporting and contradicting the final hypothesis.
  • Confidence trajectoryHow support for the hypothesis changed across rounds.
  • Final assessmentThe best-supported hypothesis and its remaining uncertainties.
  • BibliographyBibTeX for all papers consulted.

Saved to: outputs/<slug>-autoresearch.md

Workflow

watch

watch registers a saved ADS search and reports new papers each time you run watch run. With --digest, it asks the agent to summarize what is new and notable since the last check.

Usage

shell
# Add a watch (topic text becomes the ADS query)
almagest watch add "galaxy quenching"

# Add a watch with a custom ADS query
almagest watch add "IFU metallicity gradients" \
  --query "abs:metallicity abs:gradient (bibstem:ApJ OR bibstem:MNRAS) year:2024-2025"

# Watch a specific author's new papers
almagest watch add "Pillepich new papers" \
  --query "author:Pillepich,A year:2024-2025"

# List all registered watches
almagest watch list

# Check for new papers
almagest watch run

# Check + generate an AI digest
almagest watch run --digest

# Remove a watch
almagest watch remove galaxy-quenching

Output

Without --digest: a plain list of new papers since the last run (bibcode, authors, title, citation count).

With --digest: an AI-written summary of the new papers, grouped by sub-topic, with brief annotations explaining what each paper contributes.

Saved to: outputs/<slug>-digest-<date>.md

Almagest uses full ADS Solr query syntax. These work in almagest search, in watch add --query, and in the agent's tool calls.

Authorauthor:"Navarro, J"
Year rangeyear:2020-2025
Specific journalbibstem:ApJ · bibstem:MNRAS · bibstem:A%26A
Review articlesdoctype:review
Open access onlyproperty:openaccess
Software papersproperty:software
arXiv classarxiv_class:astro-ph.GA
Refereed onlyproperty:refereed

Reference

Output format

All generated documents follow the same conventions so outputs are consistent and citable.

Inline citations

Every factual claim is cited in the format:

(Navarro et al. 1997, `1997ApJ...490..493N`)

Claim labels

Claims that cannot be directly verified receive a label:

LabelMeaning
(no label)Directly supported by the cited paper.
[INFERRED]Logical inference from cited evidence — not stated explicitly in the source.
[UNVERIFIED]Could not be confirmed against the cited source.
[SPECULATIVE]Author's extrapolation, not drawn from the literature.

Provenance sidecar

Every output file gets a companion .provenance.md file that records:

  • Generation timestamp (UTC)
  • Workflow that produced it
  • All ADS bibcodes and URLs consulted

Reference

Local LLM setup

Almagest works with any OpenAI-compatible local server. Set LLM_PROVIDER=local in your .env and point it at your server.

Ollama

shell
# Install Ollama from https://ollama.com, then:
ollama pull llama3.2
ollama serve
.env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:11434/v1
LOCAL_LLM_MODEL=llama3.2
LOCAL_LLM_API_KEY=ollama

LM Studio / vLLM

Start your server and set LOCAL_LLM_BASE_URL to its base URL. Set LOCAL_LLM_MODEL to the model name as it appears in the server.

Tool-use support — models with native function calling (Qwen 2.5, Llama 3.1+, Mistral) give the best results. Models without it fall back to text-based tool calling, which is less reliable for complex workflows.
