ADS query syntax

Almagest uses the full NASA ADS Solr query syntax. These operators work in almagest search, in watch add --query, and in the agent's internal tool calls.

GoalQuery example
Keyword in abstractabs:"stellar feedback"
Keyword in titletitle:"dark matter halo"
Authorauthor:"Navarro, J"
Year rangeyear:2020-2025
Specific journalbibstem:ApJ · bibstem:MNRAS · bibstem:A%26A
Review articlesdoctype:review
Open access onlyproperty:openaccess
Refereed onlyproperty:refereed
Software papersproperty:software
arXiv classarxiv_class:astro-ph.GA
Papers citing Xcitations(bibcode:1997ApJ...490..493N)
References of Xreferences(bibcode:2005ApJ...624L..85M)
Booleanabs:"AGN" AND (bibstem:ApJ OR bibstem:MNRAS)
Combinedabs:"quenching" author:"Peng" year:2010-2020 property:refereed

Output format

All generated documents follow shared conventions so outputs are consistent and citable.

Inline citations

Every factual claim is cited using the format:

(Navarro et al. 1997, `1997ApJ...490..493N`)

Claim labels

Claims that cannot be directly verified from the cited source receive a label:

LabelMeaning
(no label)Directly supported by the cited paper.
[INFERRED]Logical inference from cited evidence — not stated explicitly in the source.
[UNVERIFIED]Could not be confirmed against the cited source.
[SPECULATIVE]The author's extrapolation beyond what the literature supports.

Provenance sidecar

Every output file gets a companion .provenance.md file that records:

  • Generation timestamp (UTC)
  • Workflow that produced it
  • All ADS bibcodes and URLs consulted

File naming

Slugs are derived from the topic using slugify(topic) — lowercase, hyphens, max 60 characters. Example: "dark matter halo formation" → dark-matter-halo-formation.

Local LLM setup

Almagest works with any OpenAI-compatible local server. Set LLM_PROVIDER=local in your .env and point it at your server. No Anthropic API key is needed.

Ollama

shell
# Install Ollama from https://ollama.com, then:
ollama pull llama3.2
ollama serve        # starts server at localhost:11434
.env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:11434/v1
LOCAL_LLM_MODEL=llama3.2
LOCAL_LLM_API_KEY=ollama

LM Studio

Start the local server from the LM Studio UI, then set:

.env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:1234/v1
LOCAL_LLM_MODEL=<model-name-as-shown-in-LM-Studio>
LOCAL_LLM_API_KEY=lm-studio

vLLM

.env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:8000/v1
LOCAL_LLM_MODEL=Qwen/Qwen2.5-7B-Instruct
LOCAL_LLM_API_KEY=token-abc123
Tool-use support — models with native function calling (Qwen 2.5, Llama 3.1+, Mistral) give the best results. Models without it fall back to text-based tool calling, which is less reliable for multi-step workflows like deepresearch or autoresearch.
