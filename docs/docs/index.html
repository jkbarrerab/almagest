Getting Started

Installation

Almagest requires Python 3.11+. Clone the repo and install it into a virtual environment:

shell
git clone https://github.com/jkbarrerab/almagest.git
cd almagest
python3.11 -m venv .venv && source .venv/bin/activate
pip install -e .

After installation almagest is on your PATH inside the virtual environment. To call it from any directory without the .venv/bin/ prefix, add it to your shell profile:

~/.zshrc or ~/.bashrc
export PATH="/path/to/almagest/.venv/bin:$PATH"

Copy the example env file and fill in your credentials. Almagest loads .env from the current directory first, then falls back to ~/.almagest/.env.

shell
cp .env.example .env

Verify your setup before running any workflow:

shell
almagest config-check
Getting Started

Quick start

Pass a topic or bibcode as the first argument to any workflow. Outputs are written to outputs/ as Markdown files with a .provenance.md sidecar.

shell
# Literature review
almagest lit "AGN feedback quenching"

# Deep multi-agent research brief
almagest deepresearch "JWST constraints on galaxy formation at z > 10"

# All papers about a specific astronomical object
almagest source "NGC 1068" --topic "integral field spectroscopy"

# Simulated peer review of a local PDF
almagest review --pdf ~/Downloads/my_paper.pdf

# Monitor a topic for new papers
almagest watch add "stellar feedback dwarf galaxies"
almagest watch run --digest

See the Workflows page for the full documentation of each command.

Extra context

Every workflow accepts --context and --context-file to inject additional instructions, bibcodes, or constraints into the agent's prompt before it starts.

Inline text
almagest lit "AGN feedback" \
  --context "Focus on low-mass galaxies (M* < 10^10 Msun) and observational evidence only"
From a file
almagest review --pdf paper.pdf --context-file my_notes.md

Use context to steer the research direction, provide bibcodes you want considered, restrict scope to a time window or instrument, or ask the agent to focus on a particular claim or method.

Tip: Save context files for recurring tasks — e.g. a review_notes.md with your standard review criteria that you pass to every almagest review run.
