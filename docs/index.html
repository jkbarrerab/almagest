for astrophysicists

Workflows

Ten slash commands.
Natural language. Your call.

Each workflow is a structured agent pipeline that searches NASA ADS, fetches papers, and synthesises findings into a citable output file.

/deepresearch

Deep Research

Multi-agent investigation across papers, web, and code. A planner delegates to parallel researchers, a synthesizer writes the brief.

/lit

Literature Review

Literature review from primary sources with consensus mapping. Foundational papers, key debates, gaps, and a recommended reading path.

/source

Source Literature

All literature for a named object (NGC 1068, M87, …) or a sky position + radius. Resolves names via SIMBAD/NED. Optional topic filter.

/review

Peer Review

Simulated peer review with severity scores and a revision plan. Accepts an ADS bibcode, a local PDF, or both.

/audit

Reproducibility Audit

Paper-to-code mismatch audit. Compares claims, parameters, and methods in the paper against any linked GitHub or Zenodo repository.

/replicate

Replication Plan

Generates a step-by-step replication plan — data access, software environment, expected outputs. Optionally executes it.

/compare

Compare

Side-by-side source comparison with agreement and conflict matrix. Useful for competing theories or methods on the same topic.

/draft

Draft

Polished paper-style draft with inline citations from findings. Full structure — abstract through conclusions — plus BibTeX.

/autoresearch

Auto-Research

Autonomous loop: hypothesize, search, measure, repeat. Iterates until the evidence converges or max rounds is reached.

/watch

Watch

Recurring monitor for new papers on a topic. Register a saved search; run it any time to get a digest of what's new.

/coauthor

Co-author Review

Invited as a co-author, the agent reads your draft and replies with an honest, friendly email — collegial feedback, not a referee report.

✦

Guide any workflow with extra context

Every command accepts --context (inline text) or --context-file (path to a markdown file) to inject your own notes, bibcodes, or constraints directly into the agent's prompt. Ask it to consider specific papers, restrict the scope, or address a particular concern.

Features

Built for astronomers

Full ADS query syntax, PDF ingestion, provenance tracking, and your choice of language model.

🔭

Full NASA ADS integration

Search with the complete ADS query syntax — author:, bibstem:, year:, doctype:review, citations(), references(). Fetch abstracts, full text, metrics, and BibTeX in one call.

📄

Local PDF review

Pass any PDF — preprint, paywalled paper, or your own manuscript — directly to the review workflow. The agent reads the full text and searches ADS for related work it may have missed.

🤖

Claude or local LLM

Defaults to Claude (Opus / Sonnet). Switch to any OpenAI-compatible local server — Ollama, LM Studio, vLLM — by changing two lines in your .env file. No code changes needed.

🗂️

Provenance by default

Every output file gets a .provenance.md sidecar recording generation time, workflow used, and all ADS bibcodes and URLs consulted. No more wondering where a claim came from.

Multi-agent architecture

Four specialised agents

Complex workflows delegate to sub-agents, each with a focused role and its own system prompt.

🔍

Researcher

Searches ADS with multiple query strategies, fetches papers, and extracts key claims with citation evidence.

⚖️

Reviewer

Evaluates methodology, statistics, and missing citations. Grades issues CRITICAL / MAJOR / MINOR / COSMETIC.

✍️

Writer

Synthesises findings into structured Markdown with inline ADS citations and a BibTeX bibliography.

✓

Verifier

Spot-checks bibcodes against ADS, validates that cited papers support their attributed claims, removes dead links.

Supported language models
Claude Opus / Sonnet (default)
Ollama (local)
LM Studio
vLLM
Any OpenAI-compatible API
Quick start

Up in three steps

1

Install

pip install almagest
2

Configure credentials

# Copy and fill in your API keys
cp .env.example .env

# ADS token: ui.adsabs.harvard.edu/user/settings/token
ADS_API_TOKEN=your_token
ANTHROPIC_API_KEY=sk-ant-...
3

Run a workflow

# Verify setup
almagest config-check

# Literature review
almagest lit "AGN feedback quenching"

# Review a local PDF
almagest review --pdf paper.pdf
Use a local LLM instead

No API key required

Run Almagest fully offline with Ollama. Pull any supported model and point the tool at your local server with two env vars.

# Start Ollama
ollama pull qwen2.5-coder:7b
ollama serve

# .env
LLM_PROVIDER=local
LOCAL_LLM_BASE_URL=http://localhost:11434/v1
LOCAL_LLM_MODEL=qwen2.5-coder:7b

Models with native tool-use support (Qwen 2.5, Llama 3.1) give the best results. All other models fall back to text-based tool calling.
